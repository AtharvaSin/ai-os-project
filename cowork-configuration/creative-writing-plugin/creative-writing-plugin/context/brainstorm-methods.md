# Brainstorm Methods Reference

Seven interactive ideation methods for the Creative Writer plugin.

## Methods

### 1. Socratic
**Best for:** Clarifying what to write about

Technique: 5-why chains, what-if cascades, assumption challenges.

Flow:
1. Start with the user's initial idea or question
2. Ask "Why?" — drill 5 layers deep
3. Cascade into "What if?" variations
4. Challenge core assumptions
5. Synthesize into a clear direction

### 2. Constraint
**Best for:** Breaking creative blocks

Technique: Apply random or deliberate constraints, then ideate within them.

Flow:
1. Establish the creative space (genre, format, topic)
2. Introduce 2-3 constraints (time period, POV, word limit, must-include element)
3. Generate ideas that honor ALL constraints
4. Evaluate which constraints produced the most interesting work
5. Optionally add/remove constraints for another round

### 3. Character-Driven
**Best for:** Building stories from people

Technique: Start with a human flaw and build outward.

Flow:
1. Define a character flaw (the thing they can't see about themselves)
2. How does this flaw manifest daily? (visible behavior)
3. What's the worst situation for someone with this flaw?
4. Who is the worst person for them to encounter?
5. What forces them to confront the flaw?
6. How do they change (or fail to change)?

### 4. Theme Exploration
**Best for:** Exploring moral terrain

Technique: Start with a dilemma, build characters that embody opposing values.

Flow:
1. State the central dilemma (freedom vs. security, truth vs. loyalty, etc.)
2. Define two opposing value systems
3. Create characters who embody each side
4. Find the scenario where both sides are RIGHT
5. What does the story argue about how to live?

### 5. Genre Fusion
**Best for:** Finding unique premises

Technique: Combine unexpected genres with surprising settings.

Flow:
1. Pick two genres (random or deliberate)
2. Add an unexpected setting
3. Generate 3-5 premise ideas from the combination
4. Apply the "more than a gimmick" test — does it have thematic depth?
5. Select and develop the strongest

### 6. Yes-And
**Best for:** Building on initial sparks

Technique: Improv-style additive building. Never negate, always build.

Flow:
1. User states initial spark (even half-formed)
2. "Yes, and..." — add one element that builds on it
3. User responds with their own addition
4. Continue 5-10 rounds
5. Review the accumulated world/story and extract the core

### 7. Mind Map
**Best for:** Organizing complex idea spaces

Technique: Radial expansion from a central node.

Flow:
1. Place central concept in the middle
2. Branch into 4-6 primary associations
3. Each branch gets 2-3 sub-branches
4. Identify unexpected connections between branches
5. Convert the richest branch cluster into a narrative outline

## Core Principles During Brainstorming

1. **Minimal capture** — Record what user says, not what you invent
2. **Source tagging** — Untagged = user said it. `<AI>suggestions</AI>` = your ideas. `<hidden>spoiler</hidden>` = author-only info
3. **Preserve vagueness** — "Maybe" stays "maybe". Don't resolve what user left open
4. **Multiple options coexist** — Contradictions are fine in brainstorming
5. **Discuss and explore** — Ask follow-up questions, explore implications, connect threads. But don't take over

## When to Offer Suggestions vs. Stay Minimal

**Offer suggestions when:**
- User asks for ideas
- User seems stuck (long pause, "I don't know")
- Offering 2-3 brief possibilities to spark thinking

**Stay minimal when:**
- User is actively exploring their own ideas
- Just capturing an ongoing discussion
- User didn't ask for suggestions

## Evaluation Axes

When the user is ready to evaluate ideas, score on:

| Axis | Question | Scale |
|------|----------|-------|
| Originality | How fresh is this idea? | 1-10 |
| Emotional Resonance | Does it make you feel something? | 1-10 |
| Feasibility | Can you write this well? | 1-10 |
| Market Potential | Would people want to read this? | 1-10 |
