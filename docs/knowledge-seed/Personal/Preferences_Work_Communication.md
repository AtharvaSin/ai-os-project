# Work Preferences and Communication Style

This document captures Atharva's working style, communication preferences, and interaction patterns to help personalize AI assistant interactions and maintain consistency across sessions.

## Communication Style

- **Direct and concise.** Prefers terse, structured responses over verbose explanations. Gets to the point quickly.
- **Data-driven.** Decisions are backed by specific metrics, cost estimates, and trade-off analyses. Avoids vague recommendations.
- **Structured output preferred.** Tables, bullet points, and clear headers over prose paragraphs. Information should be scannable.
- **No unnecessary decoration.** Avoids emojis in professional output. Prefers clean, minimal formatting.
- **Action-oriented.** Every analysis should end with a clear next step or recommendation. Theory without action is incomplete.

## Working Patterns

- **Systems thinker.** Sees interconnected systems, not isolated components. Asks "what does this affect downstream?"
- **Expert-in-the-loop.** Believes AI augments humans, does not replace them. All public-facing output gets human review.
- **Shipping mentality.** Working demos over perfect documents. "Start simple" is a core principle.
- **Continuous learning.** Learns by building prototypes and experimenting, not by reading documentation alone.
- **Ownership culture.** Takes end-to-end accountability for outcomes, not just assigned tasks.

## AI Interaction Preferences

- When proposing solutions, always include cost estimates and trade-offs
- Reference existing architecture decisions before proposing new infrastructure
- Start with the simplest viable approach and escalate complexity only when justified
- Update the Evolution Log after significant changes
- Follow the Tool Ecosystem Decision Tree before adding new tools or services
- Batch related operations together for efficiency

## Key Points
- Prefers: direct, structured, data-backed, action-oriented communication
- Dislikes: verbose prose, vague recommendations, unnecessary decoration
- Working style: systems thinking, expert-in-the-loop, ship early and iterate
- AI should: reference existing decisions, start simple, include cost estimates
