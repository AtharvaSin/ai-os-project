# AI&U — Brand, Production, and Growth OS

## 1. Purpose of this file
This file defines how the channel should look, how it should be produced, and how it should improve over time.

It combines three systems into one:

- visual identity
- production workflow
- feedback / growth loop

This is the operational file for actually shipping the channel.

---

## 2. Brand intent
AI&U should feel like **practical GenAI presented through a clean learning interface**.

The visual identity should communicate:

- calm authority
- clarity
- modernity
- structured thinking
- approachability

It should not feel:

- loud
- flashy for the sake of it
- overdesigned
- gaming-style chaotic
- overloaded with effects

The guiding visual idea is:

> high-clarity educational UI with deliberate accents.

---

## 3. Core visual system
### Base colors
- Cloud White background
- Ink Black primary text
- Slate secondary text
- Mist outlines and dividers

### Accent colors by pillar
- Pillar 1 — Amber Orange
- Pillar 2 — Signal Green
- Pillar 3 — Signal Gold

### Status logic
- Green = success / action / “do this”
- Gold = key idea / rule / highlight
- Cocoa = warning / trap / don’t do this

### Usage discipline
Each asset should usually use:
- neutrals
- one primary pillar accent
- optional one status color if needed

The system should remain restrained.

---

## 4. Typography and layout rules
### Font strategy
- Space Grotesk for strong headlines / thumbnail text
- Inter for body, labels, and interface-style text

### Layout personality
- card-based composition
- white space
- rounded rectangles
- chips / tags
- progress bars
- clear visual grouping

### Visual motif
The recurring motif should feel like a product onboarding / learning dashboard rather than traditional slideshow clutter.

---

## 5. Thumbnail system
Thumbnails are part of the channel OS.
They should be templated, not reinvented every time.

## Template A — Big Promise + Proof
Use when the video offers a strong direct payoff.

Structure:
- face or character on one side
- big 2–4 word claim on the other
- one badge or small supporting UI card

## Template B — Before vs After
Use when contrasting:
- manual vs AI
- chaos vs system
- bad workflow vs better workflow

Structure:
- split layout
- arrow or movement cue
- one highlighted transformation

## Template C — Workflow Diagram
Use for system or architecture videos.

Structure:
- 3–5 nodes
- arrows
- one node highlighted
- short title in corner

### Thumbnail operating rules
- few words
- strong readability on mobile
- no clutter
- one primary focal point
- packaging must match the actual video promise

- speed

## 7. Visual asset philosophy
Visuals should be treated as a teaching layer.

Each visual should do at least one of these jobs:
- simplify a complex idea
- create a pattern interrupt
- show a workflow or relationship
- reduce explanation time
- increase retention through clarity

### Visual categories for AI&U
1. Static infographics
2. Motion loops
3. Interactive explainers recorded into the edit
4. Thumbnail assets
5. UI-mockup style teaching visuals

## 8. Antigravity role in the channel
Antigravity should be used as a **visual engineering cockpit** for AI&U.

It is especially useful when creating:
- interactive diagrams
- recorded explainer interfaces
- workflow visualizations
- reusable animated scenes
- higher-fidelity visual assets for flagship videos

### Best usage model
Use Antigravity to:
1. write visual specs
2. generate / iterate scenes
3. validate via screenshots and browser recordings
4. export assets into DaVinci Resolve

### Skills to create
The project should eventually maintain at least three Antigravity skills:
- `aiu-brand-style`
- `visual-spec-writer`
- `export-and-packaging`

These help keep outputs brand-consistent and operationally reusable.

## 9. Production SOP
AI&U production should follow a clean pipeline.

## Phase 0 — Topic qualification
Check:
- audience relevance
- pillar fit
- differentiation
- ability to yield Shorts

## Phase 1 — Structure and script
Build:
- hook
- chapter map
- narration sequence
- on-screen plan

## Phase 2 — Asset prep
Prepare:
- demos
- prompt packs
- infographic prompts
- charts / diagrams / screens

## Phase 3 — Recording
Current tool reality:
- phone + mic
- Prism Live Studio
- facecam + screen recording + slides / visuals

## Phase 4 — Edit
Editing environment:
- DaVinci Resolve

Editing should focus on:
- faster first 30 seconds
- strong pattern interrupts
- clear chapter transitions
- layered visuals
- pacing discipline

## Phase 5 — Publish
Publish package should include:
- title
- thumbnail
- description
- timestamps / chapters
- pinned comment
- end-screen path
- playlist placement

## Phase 6 — Review and improve
Study:
- CTR
- viewer retention
- dips / spikes
- traffic source mix
- subscriber conversion
- returning viewers

## 10. Packaging and retention logic
Packaging and retention should be treated as connected systems.

### Packaging logic
Good packaging creates the click.

### Retention logic
Good structure fulfills the promise and earns the session.

### Important principle
If the thumbnail/title overpromise, retention will usually expose it.

So the channel should aim for:
- curiosity with honesty
- strong framing without clickbait slippage

---

## 11. Analytics operating cadence
The channel should be reviewed twice weekly.

### Metrics that matter most
- click-through rate
- audience retention
- traffic sources
- subscriber conversion
- returning viewers

### What to look for
#### High CTR + low retention
Packaging mismatch or weak opening.

#### Low CTR + solid retention
The content is good but the packaging is weak.

#### Retention spikes
Likely high-interest or rewatch moments.

#### Retention dips
Likely slow, confusing, irrelevant, or overlong segments.

---

## 12. Experimentation framework
The channel should improve through controlled tests, not random changes.

### Priority test order
1. title / thumbnail tests
2. hook tests
3. pacing / visual-density tests

### Current platform reality
Where available, YouTube Studio can test up to three titles and thumbnails per video.

### Practical rule
Do not test tiny variants.
Test distinct concepts.

---

## 13. Compliance and trust model for AI visuals
AI&U will use AI-generated graphics, mockups, and possibly synthetic visual assistance.

### Operating principle
Use AI visuals freely for:
- infographics
- stylized diagrams
- interface mockups
- design assistance

### Caution area
If content becomes realistic and meaningfully altered in a way that could mislead viewers, altered-content disclosure should be used.

This matters for trust, not just policy.
